# Save It — ComfyUI Custom Node

**Save It** is a powerful image-saving node for ComfyUI that gives you full control over *where*, *when*, and *how* your generated images are saved — with a clean, interactive UI built right into the node.

---

## Features at a Glance

- **One-click manual save** — save only the images you actually want to keep
- **AutoSave** — automatically save every image the moment it's generated
- **Browse & Set Save Path** — pick any folder on your computer using a native folder dialog
- **Favorite Folders** — bookmark your most-used save locations for quick access
- **Save History** — view a log of your last 50 saved files
- **Timestamp or counter filenames** — choose between sequential numbering or date-time naming
- **Multiple formats** — save as PNG, JPEG, or WebP
- **A/B Image Compare** — compare two images side-by-side directly on the node canvas
- **Absolute path support** — save to any folder on your system, even outside the ComfyUI output directory
- **Metadata preserved** — workflow and prompt data are embedded in PNG files automatically

## Installation

Install directly from ComfyUI Manager by searching for Save_It.

Alternatively:

1. Go to your ComfyUI `custom_nodes` folder
2. Open a terminal or command prompt there
3. Run: `git clone https://github.com/ialhabbal/Save_It.git`
4. Restart ComfyUI

Or manually:

1. Go to your ComfyUI `custom_nodes` folder
2. Create a folder named `Save_It`
3. Copy all files into it
4. Restart ComfyUI

## Node Inputs

| Input | What it does |
|---|---|
| **images** | The image(s) coming out of your workflow — connect this to the node |
| **original_image** *(optional)* | A second image to compare against. Only used when **Compare Mode** is ON |

## Settings (Widgets)

### AutoSave
> Toggle: `AutoSave ON` / `AutoSave OFF`

When **ON**, every image generated by your workflow is saved to disk automatically — no button click needed. The manual Save button is disabled while this is active.

When **OFF** (default), images are only previewed on the node. You decide which ones to keep by clicking **Save**.

---

### Filename Prefix
> Text field, default: `ComfyUI`

Controls both the save folder and the filename.

**Examples:**

| What you type | Where it saves | Filename looks like |
|---|---|---|
| `ComfyUI` | `output/` | `ComfyUI_00001.png` |
| `Portraits/face` | `output/Portraits/` | `face_00001.png` |
| `Projects/Chars/_` | `output/Projects/Chars/` | `00001.png` |
| `D:\MyImages` | `D:\MyImages\` | `00001.png` |

> **Tip:** Use a trailing `/_` (underscore) if you want files named with just the counter, no prefix word.

> **Tip:** You can paste in an absolute path (like `C:\Users\You\Pictures`) to save directly there.

---

### Format
> Dropdown: `PNG` / `JPEG` / `WebP`

The file format your image will be saved in.

- **PNG** — lossless, best quality, preserves workflow metadata. Recommended for keeping your work.
- **JPEG** — smaller file size, no metadata, slight quality loss. Good for sharing.
- **WebP** — modern format with good compression. Good balance of size and quality.

---

### Quality
> Slider: 1–100, default: `95`

Controls compression quality for **JPEG** and **WebP** only. Higher = better quality, larger file. PNG ignores this setting.

---

### Timestamp
> Toggle: `Timestamp ON` / `Timestamp OFF`

**OFF** (default): Files are numbered sequentially — `image_00001.png`, `image_00002.png`, etc. The counter is remembered between sessions.

**ON**: Files are named with the current date and time — `image_2025-04-29_14-32-01.png`. Useful if you want to sort by time or avoid ever overwriting files.

---

### Compare Mode
> Toggle: `Compare ON` / `Compare OFF`

Enables an interactive side-by-side image comparison viewer directly on the node. Connect a second image to the **original_image** input to use this.

**Compare modes** (switch by clicking on the node):
- **Side-by-side (horizontal split)** — drag left/right to reveal each image
- **Side-by-side (vertical split)** — drag up/down to reveal each image
- **Overlay** — blend the two images with an adjustable opacity slider
- **Difference** — highlights pixels that differ between the two images

> **Note:** When Compare Mode is ON, only the primary image (connected to `images`) is saved — the comparison reference image is not.

## Buttons on the Node

### Save
Saves the currently previewed image to disk using your chosen settings. Only works when **AutoSave is OFF**.

### Browse & Set Save Path
Opens a native folder picker dialog so you can visually navigate to any folder on your computer. After selecting a folder, you'll be asked if you want to save it as a Favorite for quick access later.

### Favorites
Opens the Favorite Folders panel where you can:
- See all your saved folder shortcuts
- Click **Set** on any favorite to instantly use it as your save path
- Add new folders manually
- Search through your favorites
- Delete favorites you no longer need

### History
Shows a list of the last 50 files you've saved with this node, including the filename, full path, and timestamp. You can clear the history from this panel.

## How Saving Works (Under the Hood)

**When AutoSave is OFF:** Images are displayed as previews in the node. When you click Save, the preview is converted and written to disk in your chosen format.

**When AutoSave is ON:** Each time the workflow runs, the image is immediately saved to your output folder. If the same workflow runs again without changes, only a preview is refreshed (no duplicate file is created).

**Saving outside the ComfyUI output folder:** If you set a path outside ComfyUI's output directory (for example, another drive), the node saves the real file there while creating a temporary copy for preview — so the image still appears correctly in the node.

---

## File Naming

Files are named in the pattern: `prefix_NNNNN.ext`

The counter (`NNNNN`) is stored in a hidden file called `.save_it_counter` inside your save folder. This means counters are **per-folder** — each folder has its own counter that picks up where it left off, even after restarting ComfyUI.

## Tips & Common Workflows

**"I only want to save my best generations"**
→ Leave AutoSave OFF. Run your workflow freely, and only click **Save** on the results you want to keep.

**"I want to save everything automatically into organized folders"**
→ Turn AutoSave ON, set your Filename Prefix to something like `Characters/Hero`, and all generations will land in `output/Characters/` named `Hero_00001.png`, `Hero_00002.png`, etc.

**"I want to compare my before/after images"**
→ Connect your original image to `original_image`, turn on Compare Mode, and use the mouse to drag the split line or switch between comparison modes on the node.

**"I save to many different folders depending on the project"**
→ Use **Browse & Set Save Path** to navigate to each folder, then add them to **Favorites**. Next time, just open Favorites and click Set.

**"I want unique filenames that sort by time"**
→ Turn Timestamp ON. Each file will be named with the exact date and time it was saved.

## Notes

- The sequential counter is stored in a hidden file called `.save_it_counter` inside your save folder. Do not delete this file if you want numbering to continue from where it left off.
- When saving as JPEG or WebP and quality is a priority, set the quality slider to 95 or higher.
- AutoSave is well-suited for long unattended runs where every generation should be kept. Manual save is preferable when reviewing results and only keeping selected images.
- Favorite Folders are saved to `favorite_folders.json` on disk and persist permanently across ComfyUI restarts. You can edit this file directly to add or remove entries in bulk.
- Save History is stored in browser local storage. It will persist across sessions but will be lost if you clear your browser's stored data.
- The Browse & Set Save Path button uses the Windows native IFileOpenDialog API via ctypes and does not require tkinter. It is currently only functional on Windows.
- Workflow metadata (prompt and extra PNG info) is embedded in files saved via AutoSave when the format is PNG, preserving the full generation parameters alongside the image.
